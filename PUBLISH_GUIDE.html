TROUBLESHOOTING — COMMON ISSUES
I get "authentication failed" or "wrong password" when pushing ▼
GitHub no longer accepts your account password for git operations. You must use a Personal Access Token (PAT). Follow Step 5b above — generate a token with repo scope, then paste it when git asks for your password. If you already tried with the wrong password, run git credential reject or clear saved credentials in your OS keychain.
The site shows a blank page or 404 at my URL ▼
This almost always means the base in vite.config.js doesn't match your repo name. The file should say base: '/cloud-devops-roadmap/'. If you renamed your repo, update that value and push again. Also wait 2–3 minutes after the Actions workflow shows green before checking the URL.
GitHub Actions workflow fails — build error ▼
Click the failed workflow run in the Actions tab → expand the "Build" step → read the error. Most common causes: (1) Node version mismatch — the workflow uses Node 20 which should work with everything. (2) Missing package-lock.json — make sure you didn't add package-lock.json to .gitignore. (3) Syntax error in a component you edited — run npm run build locally first to catch errors.
I don't see the Pages option in Settings ▼
Make sure the repository is Public (not Private). GitHub Pages is only free for public repos. Go to Settings → General → Danger Zone → Change repository visibility → Make public.
How do I add a custom domain (e.g. devopspath.com)? ▼
(1) Buy a domain from Namecheap, GoDaddy, or Cloudflare Registrar (~$10/year). (2) In your domain's DNS settings, add a CNAME record pointing to YOUR_USERNAME.github.io. (3) In GitHub → Settings → Pages → Custom domain → enter your domain → Save. (4) Check "Enforce HTTPS". Also update vite.config.js to base: '/' when using a custom domain.
Can I deploy to Vercel or Netlify instead? ▼
Yes. For Vercel: sign up at vercel.com with GitHub → New Project → import your repo → Framework: Vite → Deploy. For Netlify: netlify.com → Add new site → Import from Git → select repo → Build command: npm run build → Publish directory: dist → Deploy. Important: for both Vercel and Netlify, change base: '/cloud-devops-roadmap/' to base: '/' in vite.config.js.
The PDF doesn't upload — nothing happens when I drop a file ▼
Make sure the PDF is text-based, not a scanned image. Resumes saved as PDF from Microsoft Word, Google Docs, Canva, or Notion work perfectly. Scanned photo PDFs have no readable text. You can check by opening the PDF and trying to select text — if you can select it, it will work.
